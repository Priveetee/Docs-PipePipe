# Segundo plano, emergente, pantalla completa y cola

## Primero: cómo se abrió el vídeo

**Ajustes → Comportamiento → Acción «abrir» preferida** se aplica a enlaces de vídeo entregados a PipePipe por *otra aplicación*. No es una anulación global de los toques internos. Pruébala con un enlace abierto/compartido desde navegador o mensajería; tocar un feed de PipePipe conserva el flujo normal de ficha/reproductor principal.

En un informe, indica la ruta: toque en feed/búsqueda/canal, intención Android «Abrir con», enlace de navegador u hoja de compartir. Así no se informa un comportamiento diseñado como si fuera un ajuste roto.

## Segundo plano y emergente

Para audio que se detiene al minimizar, vídeo que se recarga al volver o cierre emergente, anota modo de reproducción, cambio de aplicación, Android y acción exacta. Son problemas del ciclo de vida del reproductor, no necesariamente de extracción YouTube.

Da una secuencia breve y repetible: URL concreta → modo principal/segundo plano/emergente → botón Inicio o app X → esperar N segundos → volver. Indica si continuó el audio, si se pausó, si volvió a cargar o si empezó en 0:00. Una recarga larga al volver puede depender de la sesión upstream; aun así se necesita la secuencia para distinguirla de un cierre.

## Pantalla completa, rotación e imagen en imagen

Indica orientación antes/después, si emergente o segundo plano estaban activos y si la reproducción se reanudó, pausó o reinició. Aclara si el vídeo estaba pausado intencionadamente antes de pantalla completa: que una pausa se reanude durante esa transición es distinto de una continuación normal.

## Cola y listas

Para cierre al encolar, falta del siguiente vídeo o reinicio, incluye orden de cola, repetición/aleatorio, autoplay y origen: búsqueda, lista o feed. Para un cierre entre dos elementos, di si el reproductor ya había hecho la transición: «pulsar Encolar mientras termina el vídeo» sirve mucho más que «la cola se cierra».

Para un vídeo de lista que vuelve al inicio tras navegar fuera y volver, incluye URL/nombre de lista, los dos elementos visitados, posición y si la lista era local o en línea.

## Qué adjuntar

No adjuntes grabaciones con notificaciones privadas. Una grabación recortada es útil si muestra selector de modo, transición y resultado. Si no, aporta URL, pasos, Android, versión, modo, orientación, autoplay/repetición/aleatorio y estado de interfaz experimental.
